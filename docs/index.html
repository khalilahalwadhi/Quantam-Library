The Problem

Every deployed tokenization system is vulnerable to quantum attack

FF1 and FF3-1 are Feistel networks. Simon's quantum algorithm exploits their architecture — one half of the block passes through unchanged each round. This isn't a key-length problem. It's structural. More rounds don't help. Longer keys don't help.

NIST withdrew FF3-1 in 2025. FF1 remains specified but has a proven exponential-time quantum attack.
💳

Permanent lifetime identifiers. Once compromised, cannot be revoked. FPE tokenization is the primary protection layer.

How FAST Works

Four operations per round — full-block transformation with no pass-through

Input: x = [x₀, x₁, x₂, ..., x_{ℓ-1}]    (each xᵢ ∈ ℤₐ)

  P1  │ x₀ ← (x₀ + x_{ℓ-1}) mod a        ← Addition (key mixing)
  P2  │ x₀ ← σ[seq[r]](x₀)               ← S-box substitution (non-linearity)
  P1' │ x₀ ← (x₀ - x_w) mod a            ← Subtraction (cross-position diffusion)
  P3  │ x ← rotate_left(x, 1)             ← Circular shift (position rotation)

After ℓ rounds, every position has been the active position exactly once.
Unlike Feistel: EVERY position is transformed. No pass-through half.

Constant-Time S-boxes

256 random permutation S-boxes generated via Fisher-Yates shuffle with AES-CTR. Lookups use subtle::ConditionallySelectable linear scan — no cache-timing leaks.

Zero AES on Hot Path

Key schedule pre-computes all S-boxes and the round sequence. Encryption/decryption uses only modular arithmetic and table lookups — no AES calls per operation.

Two Security Levels

Classical128: 2s rounds per position. Quantum128: 3s rounds (50% more) for explicit quantum resistance margin.

Quick Start

Rust

use fast_core::{FastCipher, FastKey, Domain, SecurityLevel};

let key = FastKey::new(&[0x42u8; 16])?;
let cipher = FastCipher::new(&key, Domain::Decimal, SecurityLevel::Quantum128)?;

// Encrypt: "123456789" → 9 random-looking digits
let token = cipher.encrypt(b"tweak", "123456789")?;
assert_eq!(token.len(), 9);

// Decrypt: recover the original
let plain = cipher.decrypt(b"tweak", &token)?;
assert_eq!(plain, "123456789");

Python

from fast_fpe import FastCipher

cipher = FastCipher(key=b'\x42' * 16, radix=10, security="quantum-128")

token = cipher.encrypt(tweak=b"tweak", plaintext="123456789")
plain = cipher.decrypt(tweak=b"tweak", ciphertext=token)
assert plain == "123456789"

Credit Card Tokenization

let pan = "4111111111111111";
let bin = &pan[..6];           // "411111" — preserved
let middle = &pan[6..12];      // encrypted with BIN as tweak
let last4 = &pan[12..];        // "1111" — preserved

let tokenized = cipher.encrypt(bin.as_bytes(), middle)?;
let token = format!("{bin}{tokenized}{last4}");
// → "411111XXXXXX1111" — valid 16-digit PAN format
