The Quantum FPE Crisis

Author: Khalilah Aisha al-Wadhi
Date: May 2026 · Version: 1.0
View source on GitHub

Abstract

Format-Preserving Encryption (FPE) is the invisible backbone of tokenization infrastructure across the global payments industry, healthcare systems, and telecommunications networks. Every time a credit card number is tokenized for PCI DSS compliance, every time a Social Security number is masked for HIPAA, and every time a phone number is pseudonymized for analytics — FPE is almost certainly the mechanism at work. The two NIST-standardized FPE algorithms, FF1 and FF3-1 (NIST SP 800-38G), are deployed in hundreds of thousands of production systems worldwide.

Every single one of them is vulnerable to quantum attack.

This is not a theoretical curiosity or a distant concern. The structural vulnerability is fundamental: FF1 and FF3-1 are built on Feistel networks, a construction that has a proven, devastating weakness against Simon's quantum algorithm. Unlike Grover's algorithm — which merely accelerates brute force by a quadratic factor — Simon's algorithm exploits the structure of Feistel networks to achieve an exponential speedup.

This paper presents FAST (Format-preserving Addition Substitution Transformation), an SPN-based FPE algorithm published at ASIACRYPT 2021, as the quantum-safe replacement for FF1 and FF3-1.

1. The Ticking Clock

Quantum Computers and Cryptographic Sunset

Quantum computing has moved from theoretical physics to engineering reality. IBM, Google, and a growing number of well-funded startups are building machines with increasing qubit counts and decreasing error rates. The U.S. National Security Agency (NSA) has issued guidance directing all national security systems to transition to quantum-resistant cryptography. NIST completed its Post-Quantum Cryptography (PQC) standardization process in 2024, releasing ML-KEM (Kyber), ML-DSA (Dilithium), and SLH-DSA (SPHINCS+).

Harvest Now, Decrypt Later

The most pressing threat is data being collected today for decryption tomorrow. Nation-state adversaries are stockpiling encrypted data under the assumption that quantum computers will eventually make it decryptable. For tokenization systems, this is particularly alarming:

  • Credit card PANs remain valid for 3-10 years
  • Social Security numbers are permanent lifetime identifiers
  • Medical record numbers persist for decades
  • National identity numbers are irrevocable

2. Format-Preserving Encryption: The Forgotten Attack Surface

While the cryptographic community has focused on post-quantum key exchange and signatures, format-preserving encryption has received comparatively little attention. FPE is not a niche algorithm — it is deployed at massive scale. A 16-digit credit card number must encrypt to a 16-digit credit card number. A 9-digit SSN must encrypt to a 9-digit SSN.

Both NIST-standardized FPE algorithms (FF1 and FF3-1) are built on Feistel networks. In a Feistel network, the input block is split into two halves. Each round applies a pseudorandom function to one half and XORs the result to the other, then swaps. The critical observation: one half passes through unchanged every round.

3. Why Feistel = Quantum Bullseye

Simon's Algorithm Deep-Dive

Simon's algorithm (1997) solves the hidden period problem: given a function f where f(x) = f(y) iff x ⊕ y ∈ {0, s}, find s. Classically this requires O(2n/2) queries. Simon's quantum algorithm finds s in O(n) queries — an exponential speedup.

The connection to Feistel networks was formalized by Kuwakado and Morii (2010, 2012) and extended by Kaplan, Leurent, Leverrier, and Naya-Plasencia (CRYPTO 2016). Due to the Feistel structure, algebraic relationships between inputs separated by s can be recovered in polynomial time.

The attack generalizes to any constant number of Feistel rounds. For FF1's 10 rounds and FF3-1's 8 rounds, the attack requires a polynomial number of quantum queries to completely break the cipher.

More rounds don't help. The structural vulnerability is inherent to the Feistel construction itself. Each round preserves one half unchanged — this creates the algebraic relationships that Simon's algorithm exploits.

4. FAST: A Quantum-Safe SPN

FAST (Format-preserving Addition Substitution Transformation) was published at ASIACRYPT 2021. The core insight: replace the Feistel network with an SPN that operates on the entire block in every round.

The Round Function

Each round applies four operations:

  1. P1 — Addition: x₀ ← (x₀ + xℓ-1) mod a
  2. P2 — Substitution: x₀ ← σ[seq[r]](x₀) — keyed S-box permutation
  3. P1' — Subtraction: x₀ ← (x₀ - xw) mod a — cross-position diffusion
  4. P3 — Rotation: rotate entire block left by 1

S-box Pool Generation

256 random permutation S-boxes generated via Fisher-Yates shuffle with AES-256-CTR random coins and rejection sampling for unbiased selection. Constant-time lookups via subtle::ConditionallySelectable linear scan.

Key Schedule

AES-CMAC derives KSEQ (round sequence) and KS (S-box generation) from the master key and tweak. All AES work happens once during setup — zero AES calls on the encryption hot path.

5. Security Analysis

Why FAST Resists Quantum Attacks

  1. Full-block transformation: Every round modifies the active position, then rotates. After ℓ rounds, every position has been transformed. No "pass-through" half.
  2. Bijection property: FAST implements a permutation over the full domain. No hidden period exists for Simon's algorithm to find.
  3. High differential variation: Output differences are uniformly distributed — indistinguishable from a random permutation.

Neural Cryptanalysis Verification

7. Migration Path

The fast-migrate crate provides FF1 → FAST re-tokenization:

  1. Phase 1 — Dual-Write (Weeks 1-4): Deploy FAST alongside FF1. New tokens use both. Validate in production.
  2. Phase 2 — FAST-Primary (Weeks 5-8): Switch primary to FAST. Continue FF1 detokenization for existing data.
  3. Phase 3 — Batch Migration (Weeks 9-16): Re-tokenize all stored FF1 tokens using migrate_batch.
  4. Phase 4 — FF1 Sunset (Week 17+): Remove FF1 keys. All operations use FAST exclusively.

8. Recommendations

  1. Inventory all FPE deployments — identify every system using FF1 or FF3-1
  2. Classify data lifetime — prioritize SSNs and national IDs over expiring PANs
  3. Assess harvest-now-decrypt-later risk — if data traverses interceptable networks, treat as urgent
  4. Adopt FAST with Quantum128 — the 50% round increase is cheap insurance
  5. Use batch operations — amortize key setup across millions of tokens
  6. Validate with test vectors — run cross-validation tests in your environment

9. Conclusion

The quantum threat to format-preserving encryption is not speculative — it is a proven structural vulnerability. Simon's algorithm provides an exponential-time quantum attack against FF1 and FF3-1 that cannot be mitigated by increasing round counts or key lengths.

FAST provides a practical, proven, and performant solution. Its SPN architecture eliminates the structural vulnerability. Its performance is competitive with — and in batch mode, superior to — FF1. And its migration tooling provides a clear path from vulnerable Feistel-based tokenization to quantum-safe SPN-based tokenization.

The clock is ticking. The tools to fix this exist today.
