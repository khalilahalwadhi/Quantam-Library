QCB: Quantum Ciphertext Block

Provably Q2-Secure Authenticated Encryption — Technical Whitepaper

Author: Khalilah Aisha al-Wadhi · Syncoda
Date: May 2025
Implementation: github.com/khalilahalwadhi/Quantam-Library

Abstract

This paper presents QCB (Quantum Ciphertext Block), the only authenticated encryption with associated data (AEAD) scheme proven secure against adversaries with quantum superposition access to the encryption oracle — the Q2 security model. We provide a production-grade Rust implementation with Python bindings, demonstrate correctness through comprehensive test vectors, and explain why all widely deployed AEAD modes fail catastrophically under quantum attack.

QCB was originally proposed by Bhaumik, Bonnetain, Chailloux, Leurent, Naya-Plasencia, Schrottenloher, and Seurin at ASIACRYPT 2021, and is cited in IETF RFC 9771 as the sole example of a Q2-model-secure AEAD scheme. Our implementation builds upon their theoretical framework to deliver an auditable, memory-safe, side-channel-resistant library.

1. Introduction

1.1 The Quantum Threat to Symmetric Cryptography

While post-quantum cryptography has focused primarily on replacing asymmetric primitives (RSA, ECC), the symmetric landscape presents a subtler danger. Grover's algorithm provides a generic quadratic speedup against block ciphers, conventionally addressed by doubling key sizes (AES-128 → AES-256). However, this assumes the adversary only has classical access to the cryptographic oracle.

Simon's algorithm changes everything. Discovered by Daniel Simon in 1994, it solves the hidden period problem in polynomial time on a quantum computer. When an adversary can query a cryptographic oracle with quantum superpositions — the Q2 model — Simon's algorithm becomes devastating against constructions that rely on algebraic structure.

1.3 Security Models

  • Q1 (Post-Quantum Classical): Adversary has a quantum computer but makes only classical queries. AES-256 with any standard mode is considered secure.
  • Q2 (Quantum Superposition Queries): Adversary can query the encryption oracle with quantum superpositions. Simon's algorithm applies here. Only QCB is proven secure.
  • Q3 (Full Quantum Access): Both oracles accept superposition queries. Largely an open problem.

2. QCB Construction

2.1 Tweakable Block Cipher (TBC)

QCB is built on a Tweakable Block Cipher — a block cipher that accepts an additional tweak input modifying the encryption without rekeying. We construct the TBC from AES using XEX (XOR-Encrypt-XOR):

Ẽ_K(T, X) = AES_K(X ⊕ Δ) ⊕ Δ    where Δ = AES_K(T)

XEX was chosen because it avoids the algebraic structures Simon's algorithm exploits. Unlike Gray-code-based tweak derivation (OCB), XEX derives each tweak mask through a full AES encryption, eliminating exploitable periodicity.

2.2 Tweak Encoding

tweak = domain_separator (1 byte) ‖ nonce (12 bytes) ‖ block_index (3 bytes)
CodeDomainPurpose
0x01MessageFullFull 16-byte plaintext blocks
0x02MessagePartialFinal partial block (10* padded)
0x03AdFullFull 16-byte AD blocks
0x04AdPartialFinal partial AD block
0x05TagAuthentication tag generation

2.3 Message Encryption

Full blocks:

Cᵢ = Ẽ_K((0x01, N, i), Mᵢ)

Final partial block (|Mₘ| < 16):

Mₘ* = Mₘ ‖ 1 ‖ 0···0    (10* padding)
Cₘ* = Ẽ_K((0x02, N, m), Mₘ*)
Cₘ  = first |Mₘ| bytes of Cₘ*

2.4 Associated Data Processing

AD_hash = ⊕ᵢ Ẽ_K((0x03/0x04, N, i), Aᵢ)

2.5 Tag Computation

Checksum = M₁ ⊕ M₂ ⊕ ··· ⊕ Mₘ*
Tag = Ẽ_K((0x05, N, 0), Checksum ⊕ AD_hash)

3. Security Analysis

3.1 Why XEX Resists Simon's Algorithm

Simon's algorithm finds hidden periods in functions f(x) = f(x ⊕ s). OCB's Gray-code counters create exactly this structure. XEX avoids it: each Δᵢ = AES_K(tweakᵢ) is pseudorandomly derived, providing no period for Simon to find.

3.2 Formal Security Bound

Adv^{ae}_{QCB}(q, σ, τ) ≤ (5σ² + 2q) / 2ⁿ

Where q = queries, σ = total blocks, n = 128 (AES block size). Comparable to OCB's classical bound, but valid against Q2 adversaries.

4. Implementation Details

4.1 Design Principles

  • Memory Safety: #![forbid(unsafe_code)] — zero unsafe blocks
  • Side-Channel Resistance: zeroize::ZeroizeOnDrop for keys, subtle::ConstantTimeEq for tags
  • Comprehensive Testing: 20 Rust tests + 12 Python tests covering all edge cases

4.2 Performance

QCB operates at rate-1: one TBC call per plaintext block. For m message blocks and a AD blocks:

Total AES calls = 2(m + a + 1)

6. Conclusion

QCB represents a critical advancement: the first and only AEAD with proven Q2 security. As quantum capabilities grow, the transition from GCM/OCB to QCB will become as essential as the RSA → lattice migration in public-key cryptography.
